录制一个专业的产品演示视频，展示系统的核心功能和优势，适用于：

确保以下数据已生成：

## 🎨 视觉和演示技巧

### 1. 屏幕准备

**推荐分辨率**: 1920x1080
**浏览器窗口**: 全屏或固定大小（避免录制过程中调整）
**隐藏元素**:
- 浏览器书签栏
- 不必要的浏览器插件图标
- 桌面通知

### 2. 录制设置

**录屏区域**:
- 全屏录制：适合展示整体界面
- 窗口录制：聚焦应用本身
- 区域录制：强调特定功能

**音频**:
- 使用专业麦克风（推荐 Blue Yeti、Rode NT-USB）
- 在安静环境录制
- 添加背景音乐（音量不超过 -20dB）

**鼠标指针**:
- 启用鼠标高亮（方便观众跟随）
- 移动速度适中，不要过快
- 点击时稍作停顿

### 3. 演示技巧

**语速和节奏**:
- 正常语速或略慢
- 重要功能处停顿 2-3 秒
- 避免"呃"、"啊"等口头禅

**操作流畅性**:
- 提前练习演示流程
- 准备好测试数据
- 避免卡顿和等待（可后期剪辑）

**镜头语言**:
- 使用缩放强调重要信息
- 添加文字标注或箭头提示
- 使用转场效果（简洁为主）

## 🔧 故障排查

### 常见问题和解决方案

**问题 1: 数据生成失败**
```
症状: 点击"生成并上传数据"后报错
原因: Supabase 连接问题或权限不足
解决:
1. 检查 .env 文件中的 Supabase 配置
2. 确认数据库表已正确创建
3. 检查网络连接
```

**问题 2: 图表不显示**
```
症状: 页面加载正常，但图表区域空白
原因: 数据格式不匹配或 Recharts 未正确加载
解决:
1. 打开浏览器控制台查看错误
2. 刷新页面重新加载
3. 清除浏览器缓存
```

**问题 3: 登录失败**
```
症状: 使用演示账号无法登录
原因: 账号未创建或密码错误
解决:
1. 确认已运行数据库初始化脚本
2. 检查 auth.users 表是否有演示账号
3. 重置演示账号密码
```

**问题 4: 页面加载缓慢**
```
症状: 首次加载或切换页面时较慢
原因: 懒加载组件或大量数据查询
解决:
1. 提前预加载常用页面
2. 录制前先访问一遍所有页面
3. 使用生产构建版本（npm run build）
```
