## 3. 部署到 Netlify

### 方法 A: 使用 Netlify UI（最简单）⭐

**步骤 1: 创建账号并连接仓库**
```
1. 访问 https://app.netlify.com/signup
2. 使用 GitHub 账号登录
3. 点击 "Add new site" → "Import an existing project"
4. 选择你的 GitHub 仓库
```

**步骤 2: 配置构建设置**
```
Build command: npm run build
Publish directory: dist

✅ 这些设置已在 netlify.toml 中配置，会自动读取
```

**步骤 3: 设置环境变量**

在 **Site settings → Environment variables** 中添加：

```bash
# 必需变量
VITE_SUPABASE_URL=你的Supabase项目URL
VITE_SUPABASE_ANON_KEY=你的Supabase匿名密钥

# 可选变量
VITE_AI_API_KEY=AI服务API密钥
VITE_APP_VERSION=1.0.0
```

**如何获取 Supabase 凭证？**
```
1. 访问 https://app.supabase.com
2. 选择项目 → Settings → API
3. 复制:
   - Project URL → VITE_SUPABASE_URL
   - anon public key → VITE_SUPABASE_ANON_KEY
```

**步骤 4: 部署**
```
点击 "Deploy site"
等待 2-5 分钟完成构建
访问分配的 URL (如 https://your-app.netlify.app)
```

---

### 方法 B: 使用 Netlify CLI（开发者推荐）

```bash
# 1. 安装 Netlify CLI
npm install -g netlify-cli

# 2. 登录
netlify login

# 3. 初始化并部署
netlify init
netlify deploy --prod

# 4. 设置环境变量
netlify env:set VITE_SUPABASE_URL "你的URL"
netlify env:set VITE_SUPABASE_ANON_KEY "你的密钥"
```

## 🆘 常见问题

### Q: 演示数据生成失败怎么办？

**A**: 检查以下几点：
1. 确认 Supabase 配置正确（.env 或 .env.local）
2. 检查数据库表是否已创建
3. 查看浏览器控制台的错误信息
4. 确认网络连接正常

### Q: Netlify 部署后页面白屏？

**A**: 最常见原因是环境变量未设置：
1. 在 Netlify 控制台检查环境变量
2. 确保 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY` 已设置
3. 修改环境变量后需要重新部署
4. 打开浏览器控制台查看详细错误

### Q: 视频录制时页面加载慢？

**A**: 优化建议：
1. 录制前先访问一遍所有页面（预加载）
2. 使用生产构建版本（`npm run build && npm run preview`）
3. 后期剪辑时加速或删除加载过程
4. 考虑使用本地开发服务器而非生产环境

### Q: 如何更新演示数据？

**A**: 重新运行数据生成：
1. 访问 `/demo-mode`
2. 点击"生成并上传数据"
3. 系统会自动清理旧的演示数据（学号以"2024"开头）
4. 生成新的演示数据
